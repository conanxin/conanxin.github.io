hermes-lofi-skill v0.1 is a local static web LoFi launcher and status wrapper, not a full programmatic audio control API.

It wraps the open-source LoFi Engine into a Hermes Skill with six shell commands: serve, open, stop, status, report, help.

Why volume is not included

Windows browser audio does not route through WSLg PulseAudio, so pactl cannot target only the browser stream. The only available sink is RDPSink, which controls the entire WSLg audio output — including PAR's ffplay and other Linux media players. Adjusting it would have side effects beyond the LoFi browser tab. Use the Windows Volume Mixer instead.

$cat ./why_not_publish_runtime.txt

Why runtime/dist is not published to Pages

This public page documents the local-first wrapper; the playable LoFi runtime remains local. The dist directory is 37 MB of Tone.js bundles, audio samples, and Svelte assets. Hosting it on GitHub Pages would bloat the repo, increase clone times, and conflict with the project's local-first positioning. Visitors interested in the full experience should clone the upstream project.
