# DESIGN_NOTES.md — 设计笔记

## 1. 设计目标

### 核心目标
让普通读者（无需物理背景）通过**交互**而非阅读，理解庞加莱回归的核心思想、适用范围和哲学延伸。

### 次级目标
- 页面可作为独立知识产品发布至 GitHub Pages
- 所有模拟均可实际操作，"玩"过比"看过"理解更深
- 无需外部依赖，纯前端可离线运行
- 中文中文化，适合中文读者

---

## 2. 用户理解路径

```
第一层：直觉冲击（Hero + 卡片）
  → "什么东西？" → 3张卡片给出直觉答案

第二层：玩中学（4个交互模块）
  → "试试看会发生什么" → 操作中建立直觉

第三层：边界划定（理论条件 checklist）
  → "它什么时候成立？什么时候不成立？"

第四层：延伸思考（accordion + 概念地图）
  → "它和其他思想有什么关系？"

第五层：收束（总结语）
  → 一句话带走核心洞察
```

## 3. 各交互模块解决什么理解问题

### 模块 A：有限状态回归模拟器
**解决的理解问题**：为什么状态数量有限 → 重复不可避免？

普通读者会直觉认为"系统可以一直演化下去不重复"。本模块通过有限状态网格，将"重复"的必然性可视化：只要时间足够长，跳转一定重复。

**设计决策**：使用确定性规则 `(idx * 3 + 1) % N`，保证每次运行模式可预测，适合教学演示。

---

### 模块 B：相空间接近回归
**解决的理解问题**：庞加莱回归说的是"任意接近"而非完全相同。

模块 A 的完全重复容易让读者误解为"回到一模一样"。本模块在连续二维空间演示：点只需要回到初始点附近（ε 范围内）即为一次回归。

**设计决策**：使用简化的二维环面运动（模运算），使运动具有周期性，适合可视化接近回归。真实相空间的维度远高于 2，但教学上 2D 已足够说明问题。

---

### 模块 C：墨水扩散与熵的直觉冲突
**解决的理解问题**：为什么熵增看起来不可逆，但庞加莱回归没有推翻它？

本模块将问题具体化：一滴墨水扩散后，数学上不是绝对不可能回到初始状态，但概率极低。本模块演示扩散过程，让学生直观感受"混合容易、还原难"。

**设计决策**：用随机游走模拟扩散，速度可调，有"重置"功能，便于反复操作对比。没有声称这能真实证明庞加莱回归，只作为直觉演示。

---

### 模块 D：回归时间尺度计算器
**解决的理解问题**：庞加莱回归在现实中为什么不可观测？

即使对有限系统，回归时间也可能远超宇宙年龄。本模块通过数量级估算让读者感受这个时间有多长。

**设计决策**：使用 logN/logF 滑块，T ≈ N/f 为估算公式。显示条形图对比人类寿命、文明史、宇宙年龄。避免给出虚假精确数值，仅用"toy estimate"表述。

## 4. 视觉风格说明

### 色彩系统
- **主背景**：`#0b0d1a`（深空蓝黑）
- **模块背景**：`#0f1225`（略浅蓝黑）
- **卡片/面板**：`#151829` / `#1a1e35`
- **高亮色**：紫色系 `#6c63ff`（主交互色）+ 青色 `#00d4ff`（数据/信息）
- **点缀色**：`#ff6b9d`（警告/否定）
- **文字**：`#e8eaf6`（主）+ `#8890c8`（次/说明）

### 为什么不使用黄色旧报纸风？
- 旧报纸风传递的信息密度感与本项目"现代知识产品"定位不符
- 深色宇宙感更契合庞加莱回归的宏大时间尺度主题

### 为什么不使用赛博朋克风？
- 过度赛博朋克（霓虹色、噪声动效）对知识内容是干扰
- 本项目需要清晰的认知流，不是视觉噪音

### 字体
- 系统字体栈（Segoe UI / system-ui），避免引入外部字体依赖
- 移动端自动 fallback

### 动效策略
- Hero Canvas 背景：低速粒子轨道，持续微动，营造宇宙感
- 其他区域：**零装饰性动效**
- 交互反馈：按钮点击渐变、hover 边框高亮，仅此而已
- Accordion 展开：仅箭头旋转 90°，无滑动
- 动效服务理解，不服务娱乐

## 5. 技术决策

| 决策 | 原因 |
|------|------|
| 原生 HTML/CSS/JS，无框架 | 零依赖、可离线、无构建工具 |
| Canvas 用于动画（Hero + 模块 B/C） | 性能优于 DOM 动画，适合粒子和轨迹 |
| CSS Grid 用于状态网格 | 语义清晰，响应式简单 |
| 模块化 JS（函数分区） | 方便后续维护和扩展 |
| 无外部 CDN | 隐私、安全、离线可运行 |
| 所有数值估算注明"toy estimate" | 保持科学诚实性 |

## 6. 响应式设计

- **桌面端（>768px）**：双栏布局（条件 checklist）、全尺寸 Canvas
- **平板（481-768px）**：单栏为主，Canvas 缩放
- **移动端（≤480px）**：单栏，状态网格 min-width 300px 可滚动

导航栏在移动端折叠为 hamburger 菜单。
