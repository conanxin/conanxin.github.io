## 译文正文

十年前的昨天，亚马逊（Amazon）布道师杰夫·巴尔（Jeff Barr）在亚马逊云服务（AWS）博客上发表了一篇[仅222词的短文](https://aws.amazon.com/blogs/aws/amazon_s3/)，开头写道：

> 今天早些时候，我们推出了 Amazon S3——我们可靠、高度可扩展、低延迟的数据存储服务。

在那之前，亚马逊云服务（AWS）主要致力于让开发者能够接入亚马逊的零售商店；然而，S3 与零售毫无关系，至少不是直接相关。

### AWS 的起源

正如布拉德·斯通（Brad Stone）在《一网打尽》（The Everything Store）中详细描述的那样，到21世纪初，亚马逊日益受到一个事实的制约：公司内部各个团队都由一个单一的单体技术团队提供服务，该团队必须为每个项目授权并调配资源。斯通写道：

> 与此同时，贝佐斯（Bezos）迷上了一本名为《创生》（Creation）的书，作者是史蒂夫·格兰德（Steve Grand），他开发了一款名为《Creatures》的1990年代视频游戏，让玩家可以在电脑屏幕上引导并培育一个看似有智慧的生物。格兰德写道，他创造智慧生命的方法是专注于设计简单的计算构建模块，称为原语（primitives），然后退后一步，观察 surprising behaviors 涌现。
>
> 这本书……帮助厘清了关于公司自身基础设施问题的争论。如果亚马逊想要激发开发者的创造力，它不应该试图猜测开发者可能需要什么样的服务；这种猜测将基于过去的模式。相反，它应该创造原语——计算的构建模块——然后置身事外。换句话说，它需要把基础设施分解成最小、最简单的原子组件，让开发者尽可能灵活地自由访问它们。

"原语（primitives）"模型将亚马逊的基础设施模块化，有效地将原始数据中心组件转变为存储、计算、数据库等，这些不仅可以被亚马逊内部团队按需使用，也可以被外部开发者使用：

[![stratechery Year One - 274](https://i0.wp.com/stratechery.com/wp-content/uploads/2016/03/stratechery-Year-One-274.png?resize=640%2C461&ssl=1)](https://i0.wp.com/stratechery.com/wp-content/uploads/2016/03/stratechery-Year-One-274.png?ssl=1)

中间的 AWS 层有几个关键特征：

* AWS 拥有庞大的固定成本，但规模经济效应极为显著
* 构建 AWS 的成本是合理的，因为其第一个也是最好的客户就是亚马逊自身的电商业务
* AWS 对"原语"的专注意味着它可以原样出售给亚马逊之外的开发者，从而增加规模回报，进而加深 AWS 的护城河

最后一点是双赢的：开发者可以获得企业级计算资源而无需任何前期投入；与此同时，亚马逊则可以为一组自己本身就是第一个也是最好的客户的产品获得更多的规模。

### AWS 税

说 AWS 成功完成了其使命，这是一个严重的轻描淡写：它对开发者的影响正是 AWS 负责人安迪·贾西（Andy Jassy）在他的愿景声明中所写的。斯通总结道：

> 这份文件阐述了 AWS 扩展后的使命："让开发者和公司能够使用 Web 服务来构建复杂且可扩展的应用程序"……
>
> "我们试图想象一个宿舍里的学生，他/她将拥有与全球最大公司相同的基础设施，"贾西说，"我们认为这对初创公司和小型公司来说是一个极好的公平竞争环境，让它们拥有与大型公司相同的成本结构。"

事实确实如此，过去数年中几乎所有值得注意的初创公司都是在 AWS 或其竞争对手上起步的。然而，AWS 真正影响力的衡量标准，是它如何改变了围绕开发者的生态系统，[包括风险投资](https://stratechery.com/2015/venture-capital-and-the-internets-impact/)。

对亚马逊本身的影响同样显著：去年我[详细阐述](https://stratechery.com/2015/the-aws-ipo/)了 AWS 财务数据的披露如何有效地成为了一次 Facebook 级别的 IPO，而随后的财报中 AWS 展示了规模的力量——收入增长加上利润率提高——这进一步巩固了一个事实：AWS 将在很长一段时间内成为亚马逊收入和（最终）利润的重要驱动力。Social+Capital 创始人查马斯·帕利哈皮提亚（Chamath Palihapitiya）在被问到如果只投资一家公司他会选择哪家时，[在 Quora 上回答](https://www.quora.com/Chamath-Palihapitiya-If-you-had-to-put-ALL-of-your-money-in-one-investment-today-with-a-10-year-holding-period-what-would-it-be)：

> AWS 是对计算经济的一种征税。所以无论你关心的是移动应用、消费者应用、物联网（IoT）、SaaS 等，大多数公司都将使用 AWS 而不是构建自己的基础设施。电子商务是亚马逊用来"喂狗食"（dogfood）AWS 的方式，并继续这样做，使其达到任务级水准。如果你相信软件行业长期来看是一个多万亿、十万亿的行业，那么问问自己，一家对大多数行业征税的公司会有多值钱？1%、2%、5%——这都不重要，因为数字太大了——收入、利润、利润率等等。我认为目前在公开市场上没有任何更干净的垄断可以买入。

帕利哈皮提亚所指的垄断正是我上面提到的规模效应：AWS 越大，亚马逊在 AWS 服务定价方面的优势就越大，这意味着它们可以获得更多的业务，从而进一步增加优势。最终结果是，对于除了最大的云基础公司之外的所有公司来说，这种优势加上 AWS 提供的灵活性（这在运营和财务上都至关重要），将导致一个不可避免的结论：亚马逊应该满足他们所有的基础设施需求；为此服务支付的款项就是帕利哈皮提亚所说的"税"。

然而，值得考虑的是，正如 AWS 对开发者的影响扩散到了更广泛的初创生态系统中一样，AWS 对亚马逊自身的影响似乎也远远超出了它已经相当可观的利润贡献。亚马逊可能开始时是——借用斯通的书名——"一网打尽的商店"，但它的未来是成为一系列受益于规模经济的行业的税收收集者，而 AWS 就是这种模式。

### 亚马逊电商业务的转型

长期读者会记得，我在2014年曾经历过我的[亚马逊看空阶段](https://stratechery.com/2014/losing-amazon-religion/)；正是[AWS](https://stratechery.com/2015/the-aws-ipo/)让我收回了看空言论。然而，即使在我收回看空言论时，我仍辩称我对亚马逊电商业务的看空分析是正确的：

* 亚马逊的"媒体"业务（书籍、CD、DVD 和视频游戏）提供了绝大部分"利润"，但这项业务在亚马逊总销售额中的占比正在缩小，考虑到媒体的长期趋势，其绝对值可能也会继续缩小
* "电子产品及日用百货"增长迅速，但由于所售商品的性质，利润空间相对较小

但是，如果亚马逊满足于在"电子产品及日用百货"的销售中不赚取任何利润呢？我并不是在[马修·伊格莱西亚斯（Mathew Yglesias）](http://www.slate.com/blogs/moneybox/2013/01/29/amazon_q4_profits_fall_45_percent.html)的意义上说，亚马逊"是一个由投资界某些元素为消费者运营的非营利组织"；相反，如果亚马逊电商业务的商业模式已经变成了"税收"收集呢？

想想 Costco（好市多）：去年这家批发零售商在对其超过8100万会员的1140亿美元销售额中实现了23亿美元的净收入；会员费总额为25亿美元。换句话说，Costco 11%的毛利率甚至不足以覆盖经营业务的成本；差额以及全部利润都来自对 Costco 顾客征收的"税"。

我认为 Prime 会员对亚马逊扮演同样的角色：非 AWS 业务去年产生了26亿美元的营业利润；与此同时，消费者情报研究合作伙伴（CIRP）估计亚马逊现在拥有5400万 Prime 会员，按每位99美元计算，将产生53亿美元的收入；亚马逊电商业务的盈利能力——如果有的话——差异来自对亚马逊最好的顾客征收的"税"。

事实上，我认为这种分析甚至过于狭隘：电子商务[正在不可阻挡地取代](http://www.bloomberg.com/gadfly/articles/2016-01-29/wall-street-is-still-trying-to-figure-amazon-out)美国零售业越来越多的份额，而亚马逊[占据了其中50%以上](http://www.marketwatch.com/story/amazon-will-account-for-more-than-half-of-2015-e-commerce-growth-says-macquarie-2015-12-22)的电商增长。将这一现实与 Prime 的增长结合起来，亚马逊实际上正在走向对所有零售征收一种税的路上。

同样，正如 AWS 的情况一样，这种税是消费者心甘情愿接受的：Prime 是一种超级体验，拥有优越的价格和优越的选择，它本身也喂养着规模游戏。结果是这样一个业务结构：

[![stratechery Year One - 275](https://i0.wp.com/stratechery.com/wp-content/uploads/2016/03/stratechery-Year-One-275.png?resize=640%2C461&ssl=1)](https://i0.wp.com/stratechery.com/wp-content/uploads/2016/03/stratechery-Year-One-275.png?ssl=1)

当然，这与 AWS 的结构相同——并且具有相似的特征：

* 电商配送拥有庞大的固定成本，但规模经济效应极为显著
* 建设亚马逊配送中心的成本是合理的，因为其第一个也是最好的客户就是亚马逊自身的电商业务
* 最后一点可能看起来奇怪，但实际上，亚马逊40%的销量（按单位计算）是由第三方商家销售的；这些商家中的大多数利用"亚马逊物流服务"（Fulfillment by Amazon，FBA），这意味着他们的商品存放在亚马逊的配送中心并由 Prime 覆盖。这增加了亚马逊配送中心的规模回报，增加了 Prime 的价值，并加深了亚马逊的护城河

"税"的类比延伸到了 Prime 之外；例如，亚马逊正在从这些第三方销售中抽取一部分，并从他们直接销售的商品中获得更大比例的收入。然而，效果是一致的：亚马逊正在对一个庞大的行业征收"税"，而没有人介意，因为亚马逊的规模确保了最好的价格和最好的体验。

### 物流与 Echo（智能音箱）

越来越清楚的是，亚马逊打算在物流方面重复这一模式：去年[试验了六架飞机](https://stratechery.com/2015/amazon-unveils-new-drone-video-amazon-flying-airplanes-amazon-the-logistics-company/)之后，该公司最近又[租用了20架](http://www.wsj.com/articles/amazon-finds-air-freight-partner-1457538676)来充实其私人物流网络；这还不包括[将其中国子公司注册](http://www.reuters.com/article/us-amazon-com-freight-idUSKCN0US2YW)为海运货代。毫不奇怪，正如《华尔街日报》所指出的：

> 在一份证券文件中，亚马逊首次将"为自己或第三方提供履约和物流服务的公司，无论是在线还是离线"识别为竞争对手。它称自己为"运输服务提供商"。在这两种情况下，这都是亚马逊首次在其年度报告（即10-K表）中包含此类措辞。

那么这可能如何发展呢？

首先，亚马逊本身将是这个物流网络第一个也是最好的客户，就像 AWS 的情况一样。这证明了建设一个与 UPS、联邦快递（Fedex）等竞争的物流网络所需的大量支出是合理的，大多数媒体将这些举措框定为亚马逊控制运输成本并提高可靠性的一种方式，尤其是在节假日期间。

然而，我认为认为亚马逊会止步于此是错误的：正如他们在 AWS 和电商配送方面所做的那样，我预计该公司会将其物流网络提供给第三方，这将增加规模回报，进而加深亚马逊最终的护城河。

备受关注的 [Echo 智能音箱](https://stratechery.com/2016/amazon-echo-expand-the-nest-failure-microsoft-and-slack/)也符合这一模式：所有常见的竞争对手都可以构建智能家居的各个部分；亚马逊将简单地提供枢纽，Echo 的成本就是智能家居的一种"税"。

### 一种原语化的组织

贝佐斯[著名的1997年致股东信](http://media.corporate-ir.net/media_files/irol/97/97664/reports/Shareholderletter97.pdf)表明，这种模式的根源从一开始就已经到位。具体来说，贝佐斯非常关注规模的力量：

> 我们的市场领导力越强，我们的经济模式就越强大。市场领导力可以直接转化为更高的收入、更高的盈利能力、更快的资本周转速度，以及相应地更强的投资资本回报率……我们选择优先发展增长，因为我们相信规模是实现我们商业模式潜力的核心。

同样清楚的是，贝佐斯当时并没有充分意识到他的模式将远远超出电子商务；然而，这正是亚马逊内部组织如此强大的原因。公司由多个相对独立的团队组成，每个团队都有自己的损益表（P&L）、责任和分布式决策权。斯通解释了一个早期的贝佐斯倡议（*斜体为我的强调*）：

> 他说，整个公司将围绕他所谓的"两个披萨团队"进行重组。员工将被组织成不到十人的自治小组——小到在加班工作时，团队成员可以用两个披萨饼填饱肚子。这些团队将被独立地派去解决亚马逊最大的问题……贝佐斯将一种混沌理论应用于管理，承认其组织的复杂性，通过**将其分解成最基本的组成部分**，希望 surprising results 能够涌现。

斯通后来写道，两个披萨团队最终并不适合所有地方，但正如他在[一篇后续文章](http://www.bloomberg.com/news/articles/2013-10-15/careers-at-amazon-why-its-so-hard-to-climb-jeff-bezoss-corporate-ladder)中指出的，公司仍然非常扁平，责任广泛分布。而在那些"最基本的组成部分"中，就是那些有利于规模和实验的原语。记住上面描述的贝佐斯和他的团队如何产生 AWS 想法的那段话：

> 如果亚马逊想要激发其开发者的创造力，它不应该试图猜测他们可能需要什么样的服务；这种猜测将基于过去的模式。相反，它应该创造原语——计算的构建模块——然后置身事外。

史蒂文·辛诺夫斯基（Steven Sinofsky）喜欢指出，组织往往会按照其组织结构图来"交付"产品，虽然我一开始认为亚马逊是在复制 AWS 模式，但事实证明，AWS 模式在很大程度上就是亚马逊自身的写照（就像 iPhone 在很大程度上反映了苹果公司的单一组织一样）：创造一堆原语，置身事外，然后从顶层抽取一层收益（take a nice skim off the top）。
