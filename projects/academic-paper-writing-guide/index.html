一、论文发表流程：从研究生成到论文表达

演示稿不是从“怎么写摘要”开始，而是先把研究工作拆成六个环节：方向、问题、思路、方法、实验、论文。

01

确定方向

选择研究大领域。不是盲目追热点，而是判断领域势能、问题密度和个人长期兴趣。

02

确定问题

从宽泛方向中抽出具体挑战。问题要能被表述、被比较、被验证。

03

确定思路

提出能解释问题的核心洞察。好思路要新颖，但也要让读者能跟上。

04

确定方法

把思路落到算法、模型、规则、系统或流程中。方法是思想的实现，不是思想本身。

05

实验验证

用标准数据、强基线、主辅实验和显著性检验建立可信证据链。

06

撰写论文

按读者理解路径重新组织材料，让审稿人快速看见贡献。

二、选题的智慧：热门与冷门之外

演示稿用“热门方向”和“冷门方向”对比，真正要训练的是对问题价值的判断力。

好题目的四个信号

重要问题

它不是小修小补，而是领域里反复出现、阻碍进展的关键困难。

自己感兴趣

足够难的问题需要长期投入，兴趣是穿越低反馈阶段的燃料。

即将变热

最好站在未来趋势的前夜，而不是人群已经拥挤的中心。

高风险性

真正有差异的研究常常伴随不确定性，要做好暂时不被承认的准备。

今天的 AI / LLM 语境怎么用？

不要只问“哪个模型更强”，而要问：什么评价还不可信？什么交互还不可控？什么数据污染还没被处理？什么 Agent 失败模式还没被表达清楚？

判断标准：能否形成“重要缺口 → 核心洞察 → 可检验证据 → 可复现细节”的闭环。

三、观念转变：从作者中心到读者中心

这份演示稿最重要的写作伦理，是把论文看成给读者使用的认知工具。

作者中心

我做了很多实验，所以都要写进去；我按做事顺序推进，所以论文也按这个顺序写；我懂这个领域，所以术语可以直接出现。

读者中心

读者先需要知道问题为什么重要；再需要知道现有方法为什么不够；然后才需要看我的方法和实验；重要信息要出现在读者预期的位置。

把“工作经历”重排成“理解路径”
信息呈现的优先级

图和例子负责建立直觉；表和曲线负责承载证据；正文负责组织逻辑；公式负责精确定义。不要把所有理解负担都压给正文和数学符号。

四、论文结构：每个部分都有任务

不要把论文看成章节拼装。每个部分都在替同一条中心贡献服务。

标题 摘要 Introduction 方法 实验 相关工作

标题：一句话压缩贡献

标题不是装饰，而是第一层过滤器。它应该包含问题、方法或贡献关键词，并能在一年后让人记住你的中心贡献。

摘要：问题、做法、方法、结果

摘要不是把所有细节压缩进去，而是让读者在最短时间内判断“这篇论文解决了什么、为什么值得继续读”。

Introduction：建立缺口并占据缺口

先说明领域重要，再说明现有方法面临的关键挑战，最后说明你的方法如何缓解挑战。不要把 Introduction 写成宽泛文献综述。

方法：先例子，再直觉，最后形式化

先讲 baseline 和 running example，让读者获得直觉；再解释核心思想；最后才进入符号定义、算法和复杂细节。

实验：构造可信证据链

主实验回答“是否有效”，消融回答“为什么有效”，分析实验回答“何时有效”，错误分析回答“边界在哪里”。

相关工作：建立领域地图

不要罗列 A 做了什么、B 做了什么。要把领域分成路线、假设、方法族，并说明你的贡献站在地图的什么位置。

五、方法与实验：把“想法”变成“可信主张”

方法写作降低理解难度；实验写作降低怀疑成本。

Running Example

全篇尽量用同一个例子解释 baseline、你的方法、关键差异和错误案例。例子是读者进入复杂方法的门。

Baseline First

先介绍读者已经熟悉或领域公认的 baseline，再说明你的方法改了哪里、为什么这个改动有意义。

Formalize Last

数学工具用于形式化已有直觉，不是用来制造高深感。每个符号都必须被定义，每个公式都要有语言解释。

Strong Baseline

使用公认数据集、强基线和合理评价指标。弱基线会让读者怀疑提升来自设置，而不是方法。

Auxiliary → Main

开发集上做参数与组件分析，测试集上做主结果。先辅后主，让证据逐步收束。

Significance

报告显著性、置信区间、多随机种子或人工评测一致性，让“提升”变成可检验主张。

六、把演示稿放进国际写作研究脉络

刘洋演示稿的“读者优先”思想，与国际科学写作研究高度一致。

CARS：Create A Research Space

Introduction 的任务可以理解为三步：建立领域重要性、指出现有缺口、说明本文如何占据这个缺口。它对应演示稿中“问题为什么重要、现有方法为什么不够、我的方法如何缓解挑战”。

IMRaD：科学论文的标准叙事

Introduction 回答为什么，Methods 回答怎么做，Results 回答发现什么，Discussion 回答意味着什么。刘洋的六步流程则是 IMRaD 的研究生成版。

Reader Expectations：结构决定理解

读者会根据句子、段落和章节结构判断重点。写作不是堆信息，而是把旧信息、新信息、强调信息放在读者预期的位置。

Reproducibility：可信论文的底线

今天的 AI / NLP 论文还需要报告数据、代码、模型版本、硬件、随机种子、prompt、推理参数、运行成本、限制与潜在风险。

七、LLM / Agent 论文的补充要求

2014 年的演示稿诞生于统计机器翻译时代，但底层原则在大模型时代更重要。

Prompt 透明

报告 system prompt、user prompt、few-shot 示例、解码参数和工具调用规则。

数据污染

说明 benchmark 是否可能被模型见过，是否使用新构造测试集或时间切分。

交互轨迹

Agent 系统要给出完整 trace：输入、计划、工具、记忆、失败恢复、最终输出。

成本指标

不仅比较效果，还要比较 token、时延、调用次数、人工标注成本与部署复杂度。

失败案例

好论文不只展示成功，还要说明失败边界、幻觉类型、不可控场景。

复现包

尽量提供代码、配置、数据说明、模型版本、环境和最小复现实验。

八、论文写作交互检查表

用它检查论文、项目报告、技术页面或研究型博客。勾选越多，越接近“读者可理解、证据可信、结构完整”。
