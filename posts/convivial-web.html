愉悦工具与互联网的灵魂

重读 Henry《A Website to Destroy All Websites》，思考如何让技术重新服务于人的自主性

$ curl https://henry.codes/writing/a-website-to-destroy-all-websites/

Henry 的文章标题像一句宣言：《A Website to Destroy All Websites》。但读完后你会发现，他不是要摧毁网站，而是要摧毁那种摧毁网站的互联网——那个被平台垄断、被算法驯化、被商业利益掏空的互联网。

这篇文章是一篇檄文，也是一份蓝图。它借用 Ivan Illich 的《Tools for Conviviality》，为我们提供了分析互联网异化的理论框架，并指向一个具体的解决方案：个人网站。

$ cat ./diagnosis.md

诊断：互联网为何变得"糟糕"

Henry 用诗意的语言描述了 Web 2.0 后的异化：

我们被给予了一片广阔、神圣的自我发现与喜悦、哲学与社区的领域；成千上万亩数字景观，可以在上面种植想象力的森林和草原，栽种学习的花园，探索我们制造的洞穴。

但我们不再使用互联网来创造自己，而是用来消耗自己——在注意力农场 doom-scrolling 脑腐内容，从 feed 里获取糟粕。

这段话精准地捕捉了现代互联网的病症。Henry 用表格对比了"过去"与"现在"：

过去 现在
HTTP 草原上自由漫步 在算法内容工厂里劳作
兴趣驱动的博客/论坛/wiki 被广告淹没的短视频、AI 配音
手工培育的社交网络 圈养猪仔般注意力的社交媒体
用代码表达创意与意义 为股东价值写复杂不安全的代码

Henry 强调他不是怀旧——早期互联网慢、人口少、阶层局限，"Web For All"是进步。问题在于我们失去了使用工具的"感觉"。

$ ./automobile-analogy.sh

汽车的寓言

Henry 引入汽车作为技术发展的经典案例，为 Illich 的理论做铺垫：

第一阶段：解放

汽车的到来是"获取和个人自由的启示"。人们可以走得更远，在一天中完成更多事情，将有限的时间更多地用于创造性意志。

第二阶段：奴役

但随着工业化和普及，边际效用递减：

  • 汽车不再允许人们远行，而是要求人们远行
  • 汽车要求建设维护基础设施
  • 许多人必须使用汽车才能完成日常事务
  • 全社会为基础设施纳税，但只有特定阶层能负担汽车
  • 无力负担者被贬至资金不足的公交系统

不再是服务社会的工具，社会现在开始服务汽车。

这个寓言与互联网的发展轨迹惊人地相似：从自由的工具，变成必需的基础设施，最终成为支配我们生活的力量。

$ cat ./illich-framework.txt

Illich 的框架：Tools for Conviviality

Henry 引入哲学家 Ivan Illich 的《Tools for Conviviality》（愉悦工具）作为分析框架。

技术发展的两阶段

Illich 识别出两个关键转折点：

  1. 乐观抵达期 — 工具增强人类能力，释放时间与能量
  2. 工业化窒息期 — 边际效用递减，社会开始为工具服务

激进垄断 (Radical Monopoly)

Illich 描述的概念：一种技术工具如此主导，以至于不使用它就会被排斥在社会之外。

  • 我们见过它与汽车
  • 我们见过它与互联网
  • 我们甚至见过它与社交媒体

不再是服务社会的工具，社会改变以适应工具的需求。

愉悦工具的特征

Illich 提出愉悦工具作为对立面，类似于 E.F. Schumacher《小即是美》中的"中间技术"：

  • 可持续
  • 能源高效（虽然通常劳动密集）
  • 本地优先
  • 设计 primarily 增强用户的自主性和创造力

Illich 特别提到手工工具、自行车和电话作为例子。Henry 的洞察是：互联网是设计愉悦工具的完美工坊——因为它具有巨大的互操作性和可扩展性。

$ cat ./web-industrialization.md

Web 的工业化

Henry 列举了过去 100 年的技术发展模式：

技术 初期收益 工业化代价
纺织革命 中产阶级崛起 60万工厂工人，不到2%赚取生活工资
抗生素 击败细菌感染 抗药性蔓延
太空探索 电信、成像突破 太空碎片威胁天文学
万维网 信息民主化 网络暴力、虚假信息、成瘾

Web 的工业化和商业化开发了提取用户资本的新模式：新信息表面意味着新广告表面，新媒体格式意味着剥夺所有权的新机制。

$ cat ./reconstruction-path.md

重建路径：愉悦工具的 Web 实践

Henry 用 Illich 的框架重新构想了 Web 的三个核心功能：

1. 教学与学习

现状：YouTube、TikTok、Medium、Substack 等平台因变现承诺吸引创作者，但缺乏所有权导致问题。平台失败、规则改变、付费墙——创作者和受众被迫选择。

愉悦方案：博客。HTML 默认免费访问，RSS 工作良好，加上 Webmention，阅读新想法、实验想法、与其他思想家一起发展想法从未如此容易——同时一直拥有这些内容。

2. 与朋友连接

现状：社交媒体将我们囚禁在内容监狱——为了与朋友连接，我们被迫创造内容或被算法消失，且所有创造的内容归平台所有。

愉悦方案：POSSE（Publish On your Own Site, Syndicate Elsewhere）+ ActivityPub + 微格式。生成自己的社交网络，与 Bluesky 或 Mastodon 互操作。这个网络设计用于所有权和去中心化，围绕叙事而非参与度，不受古怪科技亿万富翁的任性影响。

3. 为 Web 编程

现状：被要求用相同功能集构建 B2B SaaS 网站 n^∞ 次，答案越来越有限。标记被锁进 JavaScript 模板，样式代码沦为部署 border-radius-drop-shadow 组合的机制。

愉悦方案：HTML 和 CSS 比以往任何时候都更具生成性。CSS 工作组和浏览器实现者的辛勤工作使得基本 Web 工具可以实现难以置信的创新表达。JavaScript 比以往任何时候都更具渐进增强性，支持快速增长的浏览器 API。

2024 是 Web 代码的文艺复兴，它只要求作者的好奇心和实验动力。

我的实践：为什么建这个数字花园

Henry 的文章给了我建设的蓝图和道德理由。但归根结底，这是一个个人选择——选择用特定方式与互联网相处。

WSL2 里的选择

我的开发环境透露了选择的本质：

  • WSL2 Ubuntu — 不是 macOS，不是 Windows IDE，而是命令行优先的 Linux 环境
  • 8GB RAM + i7-6700 — 不是最新的 M3 MacBook，而是 2016 年的硬件
  • 纯 HTML/CSS — 不是 React/Vue/Next.js，而是 1990 年代的技术栈

这些选择不是能力限制，而是有意为之的约束。约束迫使你思考什么是真正重要的。

终端美学的意义

这个网站的终端风格不是装饰，而是宣言：

  • $ cat ./file.txt — 数据是透明的、可读的
  • 绿色提示符 > — 等待输入，邀请互动
  • 实时状态栏 — 展示过程而非仅结果
  • 等宽字体 — 代码即文本，文本即代码

与 Westenberg 的呼应

写完上一篇文章后，我意识到两篇文章的深层联系：

  • Westenberg 告诉我们：社区不可替代
  • Henry 告诉我们：个人网站是保留社区的最小可行方案

个人网站不是孤立的孤岛，而是连接的节点——通过 RSS 订阅、通过 Webmention 对话、通过 POSSE 向外传播。它是自治的（拥有数据），但又是互联的（参与网络）。

$ cat ./conclusion.txt

结论

Henry 在文末写道：

恢复控制感的路径是将自己从五马分尸的应用中解放出来，夺回互联网作为你可以拥有并为之自豪的工具集——或者对我们大多数人来说，为之深深羞愧。

祝你好运。

这是一篇令人振奋又令人羞愧的文章。振奋因为它指向可能，羞愧因为我们大多仍在使用那些"五马分尸的应用"。

但建设已经开始。这个数字花园是第一步，但不是最后一步。正如 Henry 所说：

开始小 — 允许自己失败，记录成长，发布失败的想法和成功的想法。
降低发布摩擦 — 不要被工具困扰，写 HTML 然后上线。
除非你想，否则不要担心设计 — 从谦逊的土壤中成长出百万个问题。
使用 IndieWeb — 通过 brid.gy 同步到外网，通过 Webmention 带回对话。
加入我们分享你所做的 — 在 personalsit.es。

这不是怀旧，不是 Luddism，而是深思熟虑的选择——选择让技术重新服务于人的自主性，而非相反。

这个网站是我的选择。你的呢？
