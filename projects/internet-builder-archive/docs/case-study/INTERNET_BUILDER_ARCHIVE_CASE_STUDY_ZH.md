# 从一张截图到一个中文策展资料库：旧互联网建设者资料库复盘

## 1. 引子：从一张旧互联网资料库截图开始

2026 年初，用户在 Telegram 发来一张 X 帖子配图。图中是一份英文资料索引，列出了大量关于互联网历史、创业公司、媒介文化的视频、纪录片、访谈和公司内部备忘录。内容涉及 Steve Jobs、Paul Graham、Bruce Lee、George Orwell、John Carmack 等人物，时间跨度从 1970 年代到 2010 年代。

那份截图里粗略估计有几十条条目，涉及视频、播客、文章、PDF、Memos 等多种格式。没有任何分类逻辑，没有中文说明，没有嵌入播放器，只有原始链接和一些标题缩写。

这不是一份可以直接使用或传播的资料。它更像是一份原始素材——需要经过识别、分类、中文化、验证、策展，才能变成一个真正有价值的资料库。

这就是"旧互联网建设者资料库"（Internet Builder Archive）的起点。

---

## 2. 项目目标

做中文化、可阅读、可验证、可维护的资料库。

这个目标有三个含义：

**中文化**：英文材料显示中文译名和中文简介；视频或音频条目支持中文介绍和嵌入播放器。

**可验证**：每条资料的来源都要经过确认。不是搬运链接，而是确认链接是否有效、embed 是否合法、内容是否值得推荐。

**可维护**：用结构化的 JSON 数据文件管理所有条目；用 Phase 报告记录每个决策；建立 staging 机制处理待确认条目，不把未完成内容伪装成已完成。

项目最终交付的是一个可以通过 GitHub Pages 直接托管的静态网站，包含可搜索的条目列表、分类筛选、五条主题路径、导览页、中文发布材料和 SVG 分享卡片。

## 3. 数据结构设计

### items.json：单一数据源

所有条目都存在一个 `data/items.json` 文件里。每条条目有一个稳定的 id（如 `iba-001`、`iba-002`），以及其他元数据字段。

**状态字段（status）** 是整个数据结构的核心，它决定了条目是否向用户展示、是否计入公开统计。

```
verified_embed    — 有可用的官方嵌入播放器（YouTube / Vimeo 等）
verified_source   — 无法嵌入，但来源确认有效（官网、档案、出版方）
staging           — 暂时不展示，需要人工复核来源或截图确认
```

**中文策展字段** 是这个项目区别于简单链接列表的核心。每条 verified 条目都有：

- `title_zh`：中文标题（如"写作的障碍"）
- `summary_zh`：一段话简介这条资料讲什么
- `background_zh`：为什么会出现这条资料（历史背景、人物背景）
- `key_points_zh`：列出 2–5 个核心观点
- `curator_note_zh`：中文策展笔记，说明这条资料对中文互联网建设者的价值
- `why_it_matters_zh`：为什么值得看
- `recommended_for_zh`：适合谁看
- `content_format_zh`：内容格式说明（纪录片/访谈/文章等）
- `reading_or_watching_guide_zh`：如果内容较难，给出阅读/观看建议

这七个字段的目的是让中文读者在点开任何一条资料之前，就知道自己在看什么、为什么值得看、应该用什么姿势看。

### paths.json：专题路径

`data/paths.json` 定义五条专题路径，每条路径包含：

- id、标题（中英文）、简介（中英文）
- `item_ids`：属于这条路径的条目 id 列表（只引用 verified 条目）
- `page_url` 和 `share_card_url`：对应静态页面和 SVG 分享卡片

paths.json 的设计让资料库从一个扁平的列表变成一个有导航结构的知识系统。用户不需要从 74 条资料里自己找方向，而是可以沿着一条主题线索系统性地探索。

### staging_review.json：待复核清单

`data/staging_review.json` 是一个轻量清单，记录所有 staging 条目的优先级、类别、原来源类型、待确认原因。它不重复 items.json 的所有字段，只存复核所需的最小信息。

## 4. 内容录入与来源检索

74 条条目从截图录入，录入过程遵循一个优先级顺序：

1. **公司内部备忘录（Memos）**：这类资料往往是最难找的，但也最有历史价值。先集中处理 10 条 Memos 的来源检索和 embed 嵌入。

2. **视频与演讲（纪录片 / 访谈）**：YouTube 和 Vimeo 的官方频道有大量公开演讲和纪录片。优先找官方 embed，避免非官方的盗录完整片源。

3. **文章与人生思考**：这类资料来源最分散，需要在不同平台（个人博客、出版方网站、学术档案）之间反复验证。

这个优先级不是拍脑袋决定的，而是根据"资料稀缺程度 × 来源可验证难度"来排序的。Memos 最稀缺但最难验证，所以优先处理；纪录片最常见所以可以靠后处理。

## 5. 中文策展深化

Phase 2A 的核心工作是"策展深化"。

最初录入的字段只有 `title_zh`、`summary_zh`、`source_url`。这套字段够让用户知道资料是什么，但不足以帮助用户决定是否值得点开。

策展深化的意思是：对于每一条 verified 条目，补全背景、观点、推荐理由和观看建议。这不是改写内容，而是增加"上下文"——帮助中文读者建立对资料的预期，降低无效点击。

一个典型的策展深化例子是 Bruce Lee 的访谈。原始信息只有"Bruce Lee 1971 年采访，38 分钟，粤语无字幕"。策展深化后变成：

- **背景**：这是李小龙在 HBO 纪录片中的罕见英文采访，当时他还未主演《龙争虎斗》，但已经是全美武术冠军。
- **核心观点**：关于武术哲学、东西方文化、自我表达。
- **策展笔记**：这段访谈展现了 Bruce Lee 的思维方式，而不只是他的武术动作。对产品经理和创业者来说，这段话里的"打破形式"哲学值得细读。
- **观看建议**：建议看中英双语字幕版本，注意他对"water"比喻的表达。

这样的策展笔记让每条资料从"一个链接"变成"一个阅读建议"。

## 6. 合法嵌入与版权边界

这个项目在 embed 问题上有一个明确的红线：**只采用官方 embed 或明确获得授权的来源**。

官方 embed 的意思是：YouTube 官方频道的嵌入、Vimeo 官方上传、PBS / BBC / Netflix 等公共教育平台的内容。这些 embed 通常在版权上没有问题，而且质量稳定。

**source-only** 条目的意思是：某些资料无法嵌入（比如某篇发表在个人博客的文章，或某份只存在于 Internet Archive 的 PDF），但来源本身是可验证的。这类条目保留 `source_url` 供用户自行访问，不强求嵌入。

对于那些"看起来像纪录片但实际上只有盗版来源"的条目，这个项目选择直接降级为 `staging`，而不是用一个不可靠的 embed 充数。

## 7. 专题路径设计

五条路径的划分逻辑不是"按格式分类"（视频/文章/Memos），而是"按主题价值分类"。

**创始人精神谱系**：聚焦如何在产品、市场和失败中活下来。Paul Graham 的文章和 Steve Jobs 的纪录片是核心条目。适合正在创业或考虑创业的读者。

**技术创业史**：从个人计算时代到互联网浪潮。John Carmack、Steve Wozniak、Kevin Kelly 的资料构成主线。适合对技术史有兴趣的读者。

**媒介、社会与自我**：广告、宣传、成瘾、记忆如何塑造现代人。收录了 Manufacturing Consent、The Century of the Self 等经典纪录片，以及 Substack 和媒介评论文章。适合对文化和传播有兴趣的读者。

**创作者心智与长期主义**：关于写作、创造、技艺和持续一生的项目。收录了 George Orwell 的写作原则、Steven Pressfield 的创作论，以及大量长期主义者的人物案例。适合个人项目者和创作者。

**组织备忘录与战略转折**：公司内部文件里的真实决策逻辑。收录了 Marc Andreessen 的备忘录、Steve Jobs 的战略决策案例、以及风投内部备忘录。适合管理层和战略思考者。

这五条路径让资料库从"列表"变成"路线图"。用户可以沿着一条线索系统性地探索，而不需要在 74 条资料里自行筛选。

## 8. 发布与传播系统

一个资料库不能只存在本地。它需要被看到、被理解、被传播。

为此，这个项目建立了一套发布与传播系统：

**guide.html**：18KB 的中文导览页，系统介绍这个资料库是什么、适合谁、怎么用、有什么特别之处。包含 15 个 Open Graph 和 Twitter Card meta，确保在社交分享时显示正确的预览卡片。

**路径静态页**：每条路径有独立的静态 HTML 页面，包含路径说明、条目列表和策展笔记。这些页面可以独立被搜索引擎收录，也可以在社交媒体上单独分享。

**SVG 分享卡片**：每条路径有一张 1200×630 的专属 SVG 分享卡片，适合在 X / 微博 / Telegram 等平台直接使用。卡片是纯 SVG 代码，不引用外部字体或图片，在任何环境下都能正确渲染。

**README 展示截图**：用 `/snap/bin/chromium` 在本地启动 HTTP 服务器并截图，生成了 6 张 PNG 截图，展示索引页、路径页、导览页的实际效果。这些截图让潜在读者在点进项目之前就能看到页面的视觉质量。

**Open Graph / Twitter Card**：主页面、导览页和所有路径页都配置了完整的 og:title、og:description、og:image、og:url、twitter:card 等 meta。确保分享到任何平台都能显示正确的预览卡片。

**sitemap.xml / robots.txt**：主站 sitemap 包含 8 个 iba 核心页面 URL，robots.txt 指向 sitemap，让搜索引擎能够发现和索引这些页面。

**短发布帖**：为每条路径生成了三种中文发布文案（X 短帖版 / Telegram 说明版 / 朋友圈版），加上 5 个可选短标题和英文简介。这让每条路径都可以作为独立内容单独发布。

## 9. 审计与降级

Phase 2M 是这个项目第一次做系统性审计。目的是在正式宣告"完成"之前，先确认数据是可靠的、页面是完整的、链接是有效的。

审计发现了 4 个需要处理的 source_url 问题。

**iba-012（Why I Write — George Orwell）**：原 source_url 是 orwell.ru，返回 404。通过检索找到了 Orwell Foundation 官方页面（orwellfoundation.com），替换后验证 HTTP 200。这条保持 `verified_source` 状态。

**iba-002（The Real Heroes Are Dead）**：lightningdoor.com 站点不存在，原博文路径已下线。无法找到可靠的替代来源，降级为 `staging`，优先级 P2。

**iba-020（Do It Now — Steve Pavlina）**：stevepavlina.com 博客路径已变更，原 URL 返回 404。Internet Archive 没有找到有效存档。降级为 `staging`，优先级 P2。

**iba-022（Why am I always late — Steve Pavlina）**：同上，来源 404，降级为 `staging`，优先级 P2。

iba-020 的移除导致了派生内容的连锁更新：creator-mindset 路径的 item count 从 10 变为 9；README、路径页、SVG 分享卡片的数量标注全部需要更新。这是 Phase 2M-C 处理的问题。

这个降级过程说明了一个原则：**可靠性优先于表面完成率**。把 4 条来源失效的条目维持在"看起来完成了"的状态，对读者没有价值；但把它们明确标记为 staging，读者就知道这些条目还需要额外工作。

## 12. 可复用的方法论

这个项目沉淀了一套可复用的工作流：

**截图识别 → 数据 schema**：从非结构化素材（截图）提取信息，建立标准化的 JSON 数据模型。每条条目有 id、状态、分类、语言、来源、元数据。

**source lookup → verified/staging**：来源检索的结果映射到三种状态（verified_embed / verified_source / staging）。不是所有资料都能立即确认为 verified；用 staging 机制处理不确定性，不把未完成内容伪装成已完成。

**中文策展字段**：七个子字段让每条资料有上下文、有推荐理由、有观看建议。这套字段是策展价值的核心，而不是链接本身。

**专题路径**：paths.json 让扁平列表变成主题路线图。每条路径只引用 verified 条目，确保路径质量可控。

**派生页面**：路径静态页、导览页、SVG 分享卡片、截图都是 items.json 的派生内容。数据源更新后，这些派生内容需要同步更新（Phase 2M-C 处理的就是这类同步问题）。

**发布材料**：短发布帖、长文、X 帖子串、OG meta 让项目可以被发现和传播。发布材料不是装饰，而是项目可维护性的证明。

**审计与降级**：定期抽查外部链接，对失效来源降级处理，不让表面数字掩盖实际问题。

**后续维护流程**：CONTENT_GUIDE 提供录入规范，staging_review.json 提供待处理清单，Phase 报告记录每个决策。

## 13. 对后续项目的启发

这个项目的方法论可以作为四类项目的模板：

**个人 Memex 资料库**：把收集的资料从"浏览器书签"变成有策展笔记的中文资料索引。每新增一条资料，都按 DATA_SCHEMA 补全字段、按 CONTENT_GUIDE 写策展笔记。

**AI agent 辅助策展工作流**：截图 → 数据录入 → 来源检索 → 中文策展 → 质量审计，整个流程可以在 agent 中实现自动化，agent 负责检索和初稿，用户负责审核和决策。

**GitHub Pages 静态知识库**：这套工具链（JSON 数据 + 静态 HTML + 专题路径 + OG meta + sitemap）完全基于 GitHub Pages，不需要后端，不需要数据库，不需要复杂部署。

**英文资料中文化项目**：对于想把英文资料引入中文世界的项目，这套流程（title_zh 翻译 + summary_zh 写作 + curator_note_zh 策展 + why_it_matters_zh 推荐理由）提供了一套可操作的模板。

## 14. 结尾

旧互联网建设者资料库从一张截图开始，经过录入、策展、路径设计、发布系统建设和系统性审计，变成了一个可上线、可搜索、可分享、可持续维护的中文资料库。

它不是完美的。7 条 staging 还需要处理，审计发现的失效链接还需要人工逐一核实，有些条目的策展笔记还有优化空间。

但它是可以正常运行的。67 条 verified 条目全部有中文策展笔记；5 条路径全部可以独立访问；整个项目通过 GitHub Pages 直接托管，不需要任何特殊基础设施。
