# 《后来礼物 / Aftergift》产品说明书

> 版本：1.0（Phase 1E 归档版）
> 状态：静态原型 · 已完成 Phase 1 开发
> 线上预览：`https://conanxin.github.io/drafts/aftergift-prototype/`

---

## 1. 产品名称

- **中文名称**：《后来礼物》
- **英文名称**：Aftergift
- **一句话定位**：给一件旧礼物，一个体面的下一站。

---

## 2. 产品背景

在恋人、夫妻、朋友、家人、同事等各种关系中，人们经常互赠礼物。

但当关系发生重大变化——结束、破裂、疏远——这些礼物有时会变成一个尴尬的难题：

- **留着难受**：每每看见都会唤起回忆和情绪
- **扔掉可惜**：物品本身仍有价值，丢弃不环保
- **转卖缺少语境**：在普通二手平台上，这件礼物的来历往往说不出口

《后来礼物》应运而生：用户可以把礼物的故事写下来，然后选择合适的方式让它体面地流转——出售、交换、赠送、捐出，或者只讲完这个故事。

---

## 3. 核心价值

| 价值 | 说明 |
|------|------|
| 旧物整理 | 帮助用户处理关系结束后难以处置的礼物 |
| 情感告别 | 通过书写故事，完成与旧物的正式告别仪式 |
| 匿名故事 | 不暴露真实身份，用代称讲出礼物背后的关系 |
| 温柔流转 | 平台不鼓励竞价、控诉或曝光，而是促成体面的流转 |
| 反报复 | 明确禁止曝光、羞辱、报复、网暴；聚焦礼物而非审判人 |

---

## 4. 目标用户

- 刚结束一段感情，希望给旧礼物一个体面去处的人
- 想处理旧礼物但不想扔掉、想让它继续发挥价值的人
- 对有故事的旧物感兴趣、喜欢收藏和交换的人
- 关注情感整理、断舍离、循环经济的用户群体
- 想找一个安全空间倾诉关系结束故事的人

**非目标用户**：
- 想要曝光前任、发布报复性内容的人（平台明确禁止）
- 寻找二手交易最低价的用户（这不是产品定位）

---

## 5. 核心功能

### 5.1 礼物故事流

- 展示所有公开的礼物卡片，每张包含：礼物名称、类型、关系、情绪标签、处理方式、一句话故事、价格/交换意向
- 初始展示 8 条，点击「阅读更多」加载更多
- 支持按处理方式筛选：全部 / 已收藏 / 出售 / 交换 / 赠送 / 捐出 / 只展示故事
- 无结果时显示温柔空状态

### 5.2 礼物详情 Modal

- 点击卡片打开详情 Modal
- 展示完整故事（600-2000 字中文）
- 安全提示（平台声明不参与真实交易）
- 按钮：我想带走它 / 我想交换 / 收藏故事 / 举报隐私问题
- 所有按钮有 UI 反馈（Toast），但无真实功能

### 5.3 发布礼物表单

- 字段：礼物名称、类型、关系、处理方式、情绪标签、价格/交换意向、一句话故事、完整故事、是否匿名
- 情绪标签（单选）：放下 / 遗憾 / 感谢 / 释怀 / 重启 / 纪念 / 治愈 / 平静
- 实时字数统计（一句话 100 字上限，完整故事显示字数）
- 完整故事 < 50 字时显示温柔提示
- 表单提交后前端临时新增卡片（无后端存储）

### 5.4 故事预检（Phase 1C）

- 点击「检查故事是否适合公开」触发
- 纯前端本地规则检测，不上传数据
- 三级输出：适合公开 / 建议修改后公开 / 不建议直接公开
- 检测：身份信息、报复性表达、可识别关系对象

### 5.5 AI 编辑建议（Phase 1D）

- 标题：「发布前的温柔检查」
- 说明文案：这些建议只在你的浏览器本地生成，不会上传你的故事
- 点击「运行温柔检查」输出：
  - 总体风险等级（适合公开 / 建议修改后公开 / 不建议直接公开）
  - 身份信息风险条目
  - 报复/攻击性表达风险条目
  - 可识别关系对象风险条目
  - 匿名化建议卡片（9 条规则）
  - 故事完整度建议（5 项）
- 支持一键复制所有匿名化建议

### 5.6 自动匿名化建议（Phase 1D）

9 条规则，每条包含：原表达类型 / 为什么有风险 / 温和替代表达建议

| 类型 | 建议改法 |
|------|---------|
| 真实姓名 | 那个人 / TA / 我曾在乎的人 |
| 手机号码 | 后来我们不再联系了 |
| 微信号 | 我们后来失去了联系 |
| QQ号 | 我们后来失去了联系 |
| 邮箱地址 | 我们后来失去了联系 |
| 报复性词汇 | 这段关系让我失去了信任 |
| 诅咒表达 | 把感受写出来，但不要用伤害性的语言 |
| 可识别地点 | 那个人工作的地方 / 我们曾住在同一座城市 |
| 住址信息 | 我们曾经住得很近 / 后来搬到了不同的地方 |

### 5.7 故事完整度检查（Phase 1D）

5 项检测，温柔引导用户补全故事：
- 字数 < 50 字
- 缺少礼物来历描述
- 缺少礼物曾经的意义
- 缺少告别理由
- 缺少对下一站的期望

### 5.8 收藏故事（Phase 1D）

- 心形按钮：卡片右上角 + Modal 内
- localStorage 持久化，同浏览器永久保存
- 筛选标签「已收藏」，无收藏时显示温柔空状态
- 按钮状态：空心（未收藏）/ 实心珊瑚色（已收藏）

### 5.9 阅读更多（Phase 1D）

- 初始展示 8 条（最多展示 100 条）
- 筛选切换时重置已展示数量
- 「阅读更多故事（还剩 N 件）」按钮

### 5.10 移动端优化（Phase 1D）

- textarea 自动增高
- 一句话故事：n/100 字数计数
- 完整故事：n 字计数 + <50 字温柔提示
- 预检按钮、预览按钮手机端全宽
- 确认 Checkbox 标签自动换行
- 360px 极窄屏专项适配
- body overflow-x: hidden 防横向滚动

---

## 6. 非目标（产品边界）

| 非目标 | 说明 |
|--------|------|
| 不是前任曝光平台 | 禁止真实姓名、手机号、地址、社交账号 |
| 不是情感审判平台 | 禁止侮辱、羞辱、报复、诅咒内容 |
| 不是真实交易平台 | 原型不处理支付、物流、资金托管 |
| 不是后端产品 | 当前为纯静态原型，无数据库、无用户系统 |
| 不处理真实支付 | 未来 Phase 4 才考虑担保交易 |

---

## 7. 产品气质

- **色调**：暖米白、浅灰、淡粉、暖棕；不阴暗、不刺眼
- **文案**：温柔、克制、干净、有一点伤感
- **交互**：大量留白、卡片式展览标签风格
- **安全提示**：明确但不吓人，柔和中带坚定
- **禁止**：报复、羞辱、网暴、猎奇传播

---

## 8. 数据现状

- 示例数据：14 条虚构礼物故事
- 关系类型：前任、挚友、夫妻、家人、同事等
- 礼物类型：家居装饰、数码、书籍、配饰等
- 所有发布者：`anonymous: true`
- 无真实人物姓名、手机号、地址、社交账号
