# 用户路径文档

> 本文档记录《后来礼物 / Aftergift》核心用户路径。每条路径包含：用户目标、操作步骤、系统反馈、可能风险、当前原型处理方式。

---

## 路径 1：浏览礼物故事

**用户目标**：随意浏览他人的礼物故事，找到感兴趣的内容

**操作步骤**：
1. 打开页面，自动加载礼物故事流（初始 8 条）
2. 向下滚动浏览卡片
3. 点击「阅读更多故事」加载更多

**系统反馈**：
- 卡片依次展示：名称、类型、关系、情绪标签、处理方式、一句话故事、价格
- 阅读更多按钮显示剩余数量，全部加载后按钮消失

**可能风险**：
- 用户看到报复性内容 → 平台禁止报复内容，故事预检拦截

**当前原型处理**：纯前端静态展示，无后端内容审核

---

## 路径 2：按处理方式筛选故事

**用户目标**：只查看自己想看的那类礼物（出售/交换/赠送/捐出/只展示故事）

**操作步骤**：
1. 点击筛选标签（如「出售」）
2. 列表自动过滤，显示对应礼物
3. 点击「全部」恢复完整列表

**系统反馈**：
- 当前选中标签高亮（`aria-selected="true"`）
- 筛选时重置「阅读更多」计数
- 无结果时显示空状态温柔提示

**可能风险**：
- 无筛选结果时用户不清楚 → 空状态文案引导

**当前原型处理**：前端 JS 筛选，筛选时重置 `displayedCount`

---

## 路径 3：查看礼物详情

**用户目标**：阅读完整故事，了解礼物背后的关系

**操作步骤**：
1. 点击任意礼物卡片
2. Modal 打开，展示完整故事（600-2000 字）
3. 阅读安全提示和操作按钮
4. 点击右上角 X 或按 Escape 关闭

**系统反馈**：
- Modal 从底部滑入（移动端）/ 居中（桌面端）
- 聚焦移到 Modal 关闭按钮
- 身体滚动锁定

**可能风险**：
- 故事内容包含可识别信息 → 举报按钮供用户反馈

**当前原型处理**：Modal 内置安全提示，举报按钮触发 Toast

---

## 路径 4：收藏 / 取消收藏故事

**用户目标**：把打动自己的故事收藏起来，之后可以随时找到

**操作步骤**：
1. 点击卡片右上角心形按钮，或在 Modal 内点击「收藏故事」
2. 按钮变为实心珊瑚色，显示 Toast「已收藏这个故事」
3. 再次点击取消收藏

**系统反馈**：
- 心形按钮状态切换（空心 ↔ 实心）
- 状态写入 localStorage，刷新页面不丢失
- 可通过「已收藏」筛选标签查看所有收藏

**可能风险**：
- localStorage 满或不可用 → 静默失败，不阻塞用户

**当前原型处理**：`localStorage.setItem('aftergift_favorites', JSON.stringify(favorites))`

## 路径 5：发布一件礼物

**用户目标**：把自己的旧礼物故事发布到平台

**操作步骤**：
1. 点击「发布一件礼物」按钮，滚动到发布区
2. 填写：礼物名称、类型、关系、处理方式、情绪标签、价格、一句话故事、完整故事
3. 可选：点击「运行温柔检查」审查内容
4. 可选：点击「预览发布效果」查看卡片样式
5. 勾选 3 项发布前确认
6. 点击「发布礼物」

**系统反馈**：
- 新卡片出现在列表最顶部
- Toast 显示「礼物已发布，愿你与它好好告别」
- 表单重置为空

**可能风险**：
- 用户发布包含真实身份信息的故事 → AI 编辑建议拦截 + 发布确认
- 用户误发布报复性内容 → 发布确认拦截 + 预检提示

**当前原型处理**：前端规则检测 + 3 项确认，发布到前端数组（无后端持久化）

---

## 路径 6：运行温柔检查 / AI 编辑建议

**用户目标**：在发布前检查故事内容是否适合公开，获得匿名化改写建议

**操作步骤**：
1. 在发布表单填写一句话故事或完整故事（建议两个都填）
2. 点击「运行温柔检查」
3. 阅读输出结果：
   - 总体风险等级
   - 身份信息风险（若有）
   - 报复性表达（若有）
   - 匿名化建议卡片
   - 故事完整度建议
4. （可选）点击「复制匿名化建议」将建议复制到剪贴板

**系统反馈**：
- AI Review Panel 展开，显示各类检测结果
- 每条匿名化建议含：类型标签 / 原表达 / 风险原因 / 建议改法

**可能风险**：
- 规则无法拦截所有规避手段（如 Base64 编码、零宽字符）
- 规则无法检测语义层面的高级报复内容

**当前原型处理**：纯正则规则，Phase 2 需要接入 AI 审核 API

---

## 路径 7：预览发布效果

**用户目标**：在发布前看到自己的卡片在列表中的样子

**操作步骤**：
1. 填写必填字段（名称、类型、处理方式、一句话故事）
2. 点击「预览发布效果」
3. Modal 展示预览样式，包含完整故事

**系统反馈**：
- Modal 顶部有「这是别人将看到的样子」温柔提示
- 预览包含：卡片标题、类型、关系、情绪、处理方式、价格、完整故事

**可能风险**：
- 预览不做内容审核，用户可直接预览包含违规信息的故事草稿

**当前原型处理**：预览仅读取表单数据，不调用审核规则

---

## 路径 8：阅读更多故事

**用户目标**：看完初始 8 条后，想看更多

**操作步骤**：
1. 在礼物列表底部点击「阅读更多故事（还剩 N 件）」
2. 系统加载下一批卡片（再展示 8 条）
3. 重复直到全部加载完毕

**系统反馈**：
- 按钮文字实时更新剩余数量
- 全部加载完毕后按钮自动消失

**可能风险**：
- 数据量大时前端性能 → 当前限制最多展示 100 条

**当前原型处理**：`displayedCount += INITIAL_DISPLAY`，最多 100 条

---

## 路径 9：通过手机浏览完整故事

**用户目标**：在手机上浏览、筛选、收藏和发布

**操作步骤**：
1. 在手机浏览器打开页面
2. 使用筛选标签浏览
3. 点击卡片查看详情
4. 收藏感兴趣的故事
5. 滚动到发布区填写表单

**系统反馈**：
- 移动端布局单列
- Modal 从底部滑出
- 表单按钮全宽
- 无横向滚动

**可能风险**：
- 小屏幕（360px）阅读体验 → 极窄屏专项适配

**当前原型处理**：360px 断点适配，emotion-tag 字号优化
