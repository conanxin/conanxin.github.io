## 交互流程

### 用户旅程 A：浏览型
1. 进入页面 → 阅读 Hero 区核心论点
2. 向下滚动 → 浏览母题标签
3. 点击感兴趣母题 → 作品网格实时筛选
4. 点击作品卡片 → 展开详情（含 quote、context、关联母题）
5. 继续浏览或切换母题

### 用户旅程 B：探索型
1. 进入页面 → 跳过 Hero（或快速浏览）
2. 直接点击"信件旅程时间线"
3. 点击某个阶段（如"等待"）
4. 查看该阶段对应的作品高亮
5. 点击进入作品详情 → 发现关联母题 → 跳转到关系图谱
6. 在图谱中拖拽探索 → 发现意料之外的作品关联

### 用户旅程 C：主题型
1. 进入页面 → 直接查看"阅读路线推荐"
2. 选择一条路线（如"从悲伤到治愈"）
3. 按顺序浏览路线中的作品
4. 每部作品可标记"已读"（localStorage 记录）
5. 完成路线后显示进度和推荐下一条路线

## 关键交互细节

### 母题筛选
- 默认状态："全部"选中，显示所有作品
- 点击单选：仅显示该母题作品，其他标签置灰
- 多选模式：按住 Shift 或点击"多选"切换后，可选择多个母题，作品取并集
- 筛选过渡：卡片以 fade + scale 动画出现/消失，时长 300ms

### 作品卡片展开
- 点击卡片 → 从卡片位置展开模态框（非全屏，桌面占 60% 宽度）
- 模态框内容：标题、年份、媒介、母题标签、letter_desc、quote、context
- 关联作品：底部显示"同母题其他作品" 3 个快捷入口
- 关闭：点击遮罩、点击 X、或按 ESC

### 时间线交互
- 7 个阶段节点均匀分布
- hover 节点：显示阶段名称 + 1 个代表作品缩略图
- 点击节点：下方展开该阶段的"作品池"（横向滚动卡片）
- 当前激活节点以颜色高亮，已完成阶段用实线连接，未完成用虚线

### 关系图谱
- 使用 D3.js 力导向图（或轻量级替代方案）
- 节点：作品（圆点）、母题（方形）、主题（菱形）
- 连线粗细表示关联强度（1-3 个母题共享）
- hover 节点：高亮所有关联边和节点，其他淡化
- 支持 pinch 缩放（移动端）和滚轮缩放（桌面）

## 移动端适配建议

1. **母题导航**：使用横向滚动容器（overflow-x: auto），隐藏滚动条，支持 touch 滑动
2. **作品卡片**：增大点击区域（≥48px），卡片内文字最小 14px
3. **时间线**：垂直布局，节点在左侧，内容在右侧，避免水平挤压
4. **关系图谱**：简化节点数量（默认仅显示作品+母题），提供"展开全部"按钮
5. **模态框**：移动端全屏显示，底部固定关闭按钮
6. **性能**：关系图谱在低端设备上降级为静态 SVG 网络图
7. **字体**：移动端正文最小 16px（避免 iOS 自动放大输入框）
