# Learning UX Design Notes — Phase 5D

## 设计目标
把 How2AI 中文课程导览从"课程资料页"升级为"可自学课程产品"，在保持无版权侵权的原则下，提供真正适合中文学习者的沉浸式学习体验。

---

## 一、三种学习模式设计

### 为什么需要学习模式？
不同学习者的目标差异很大：
- 通识学习者：只需要建立概念框架，不想被论文细节淹没
- 论文精读者：想通过论文训练研究能力，需要结构化阅读方法
- 项目实战者：已经有研究想法，需要快速找到方向和资源

### 设计实现
点击模式卡片 → 高亮 + localStorage 保存 → 全局状态更新
- `how2ai_mode` key 保存当前模式
- Hero 进度区显示当前模式标签
- 论文阅读区根据模式筛选推荐论文

### 重置机制
提供"重置学习模式"按钮，清除 localStorage 并恢复未选中状态。

## 二、每节课 Learning Output 设计

### 为什么需要？
传统课程页面只提供"读了什么"，不提供"学完能做什么"。Learning output 把被动阅读转化为主动产出：

### 字段定义
```json
{
  "learning_output": "能解释 How2AI 与传统 AI 课程的区别，并画出课程四大模块地图。"
}
```

### 设计原则
- 动词开头（能解释 / 能分析 / 能设计 / 能比较）
- 具体可验证（不是"了解多模态 AI"而是"能为一种新模态画出数据→表征→评估流程"）
- 与课程研究项目连接

---

## 三、七角色论文阅读工作台设计

### 为什么需要？
七角色论文读法是课程的核心方法论，但 Phase 3 只实现了"展示"。工作台把它变成真正可操作的工具。

### 七个视角设计
1. **Peer Reviewer**：评审视角，培养批判性思维
2. **Archaeologist**：考古视角，理解研究脉络
3. **Academic Researcher**：后续研究视角，找到新问题
4. **Industry Practitioner**：工业视角，评估实际价值
5. **Hacker**：快速原型视角，训练动手能力
6. **Private Investigator**：作者路径视角，理解研究动机
7. **Social Impact Assessor**：社会影响视角，培养责任感

### 交互流程
选择论文 → 查看元信息（难度/路径/推荐顺序）→ 选择角色 → 读取问题 → 写笔记 → 保存 → 导出 Markdown

### 数据持久化
- localStorage key: `how2ai_workbench_notes`
- 数据结构: `{paperId_role: "笔记内容"}`

## 四、项目进度系统设计

### 为什么需要？
MIT MAS.S60 的项目节奏（Proposal → Midterm → Ablation → Final）是课程骨架。提供一个独立的进度追踪系统，让学习者可以把课程学习和自己的研究项目绑定。

### 十个里程碑
1. 选题方向
2. Proposal 草案
3. Literature Review
4. 数据与任务定义
5. Baseline
6. Midterm Report
7. Error Analysis
8. Ablation Study
9. Final Presentation
10. Final Report

### 进度可视化
进度条 + 百分比 + "X/10 已完成"文字描述

### 导出
Markdown 格式，包含勾选状态 + 下一步待办事项

## 五、为什么降低首页信息密度

### 问题诊断
Phase 1–4 的主要问题：
- 首页一次性展示 27 个 session 卡片，每个卡片展开内容多
- 阅读区一次性展示 32 篇论文
- 来源链接区默认展开，占用大量垂直空间
- 移动端首屏压力巨大，需要滚动很久才能看到交互元素

### 解决方案
- **课程 session 卡片**：默认只显示一句话摘要 + 标题 + 标签；点击"展开"才显示完整内容
- **官方阅读清单**：默认折叠，只显示统计数字和"展开"按钮
- **来源链接**：默认折叠
- **术语表**：默认只显示 12 个热门术语（搜索后展开相关结果）
- **阅读清单**：按难度和路径筛选，不一次性展示全部

## 七、阅读难度与推荐顺序

### 三级难度
- 🟢 **入门（beginner）**：不需要太多背景知识，适合初学者
- 🟡 **进阶（intermediate）**：需要一定 ML/AI 基础
- 🔴 **高级（advanced）**：前沿研究，适合有经验的研究者

### 四条学习路径
- 快速通识
- 论文精读
- 项目实战
- 全部

### 推荐顺序
1–32 排序，每篇论文都有推荐的阅读顺序数字。
