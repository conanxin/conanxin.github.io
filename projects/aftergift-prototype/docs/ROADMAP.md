## Phase 2：后端 MVP

**目标**：从原型升级为可运行的数据驱动产品

### 核心技术

- **后端**：Python FastAPI 或 Node.js Express
- **数据库**：PostgreSQL（主数据）+ Redis（缓存/队列）
- **ORM**：SQLAlchemy / Prisma
- **AI 审核**：OpenAI Moderation API 或百度文本审核

### 功能清单

| 功能 | 说明 |
|------|------|
| REST API | 礼物 CRUD（发布/查看/删除/筛选/分页）|
| 用户系统 | 可选匿名注册（手机号 HASH），随机匿名昵称 |
| 发布审核 | AI 审核 + 高风险人工复审 |
| 收藏同步 | 登录用户跨设备收藏同步 |
| 举报处理 | 用户举报 → 管理员后台 → Telegram Bot 通知 |
| 管理员后台 | Web 界面处理举报、复审高风险故事 |
| 分页 API | 支持游标分页（cursor-based），避免一次加载全部数据 |

### 验收标准

- [ ] API 响应时间 P95 < 200ms
- [ ] AI 审核覆盖率 100%（所有新发布内容）
- [ ] 管理员复审队列 24 小时内处理完毕
- [ ] 无用户数据泄露（手机号 HASH 存储）

## Phase 3：真实审核层

**目标**：建立完善的内容治理体系

### 功能清单

| 功能 | 说明 |
|------|------|
| 多级风险分级 | P0（直接封禁）/ P1（人工复审）/ P2（通过但提示）/ P3（通过）|
| AI 审核升级 | 接入百度/阿里云中文内容安全，增强语义理解 |
| 申诉机制 | 用户对被拒内容可提交申诉，48h 内处理 |
| 自动匿名化 | AI 生成替代表达建议，不只是正则匹配 |
| 图片审核 | 支持礼物图片上传，图片同步接入 AI 审核 |
| 敏感词实时提示 | 发布时实时高亮敏感词（不上报，纯本地）|

### 技术升级

- 引入消息队列（Redis Streams / RabbitMQ）处理审核任务
- 图片存储：Cloudflare R2 / AWS S3 + CDN 加速
- 数据库：读写分离，主库写入，从库查询

## Phase 4：交易 / 交换机制

**目标**：让礼物真正流转起来

### 功能清单

| 功能 | 说明 |
|------|------|
| 交易意向 | 用户表达「想要这个」→ 发布者收到通知（匿名）|
| 交换匹配 | 基于「你想要」+「我有」，推荐可交换的礼物对 |
| 物流信息 | 可选填写快递单号（双方可见，匿名转发）|
| 线下交换会 | 基于城市的线下交换活动报名（Phase 5）|
| 担保支付（可行性研究）| 研究接入微信支付 / 支付宝担保交易的可能性 |
| 信用体系 | 基于历史流转成功率的轻量信用（不涉及真实身份）|

### 安全考量

- 交易版块必须有实名认证或芝麻信用等第三方信用背书
- 涉及资金必须接入持牌支付渠道
- 平台不出入金，只做担保

## Phase 5：社区与移动端

**目标**：从工具升级为社区

### 功能清单

| 功能 | 说明 |
|------|------|
| PWA 支持 | 离线缓存、添加到主屏幕、推送通知 |
| 匿名私信 | 读者向发布者发送匿名私信（平台转发，不暴露身份）|
| 温和评论 | 无实名、无排序（按时间）、无点赞（避免社交攀比）|
| 收藏故事集 | 将多个故事编入自己的收藏夹（类似 Pinterest Board）|
| 城市活动 | 城市礼物交换会报名与管理 |
| 关系旧物展 | 策展工具：用户可创建主题展览（如「十年异地恋礼物」）|
| 社区规范 | 社区自治规范，由用户共同维护 |

### 技术升级

- 移动端：React Native 或 Flutter（原生体验）
- 实时通知：WebSocket 或 SSE
- CDN：Cloudflare（全球加速）+ 图片优化

## 长期愿景

- 让「告别」变成一种有仪式感的集体行为
- 连接情感整理与循环经济
- 成为「礼物故事」这个新品类的定义者

---

## 风险与不确定性

| 风险 | 影响 | 应对 |
|------|------|------|
| 报复性内容大量涌入 | 平台被用于网络暴力 | Phase 2 强审核 + Phase 3 AI 升级 |
| 无人使用（冷启动）| 没有故事就没有流量 | 种子用户邀请 + 早期内容补贴 |
| 支付资质门槛 | 担保交易需要支付牌照 | 轻量交换版（免费）先行，支付版与合作方探索 |
| 监管风险 | 礼物故事可能被认定为情感咨询 | 明确平台定位，不提供情感建议，仅处理物品流转 |
