## 2. 设计理由

### 为什么是"研究能力栈"而不是"8 篇文章"
原帖本身就有一个隐含的栈：选择问题 → 升级输入 → 写作 → 紧循环 → 看输出 → 跨领域 → 找同伴 → 长期。把 8 段翻译写成纯长文，等于把读者放回"看起来像研究者"的状态——而本页面要让读者一眼看见**能力的可训练性**。所以：
- 颜色编码：8 层 8 个色块，让层与层之间的关系在视觉上像乐高而不是金字塔。
- 8 个 component 都是**可操作的训练动作**（预测、记录、复盘、勾选 7-day、复制 log），不是名词。

### 为什么每个 component 都用"显示状态可持久化"
- 主题切换 / 7-day 勾选 / log 输入 — 全部走 localStorage。
- 这样页面既是"可读的整理"也是"可用的工作台"。读者下次回来发现自己 Day 2 已经勾完 Day 3 没勾，会自我驱动继续。

### 为什么双主题 (research notebook / dark lab)
- notebook = 米黄 + 衬线 + 红色高亮（长期阅读、笔记感）。
- dark lab = 深底 + 琥珀强调（深夜 / 长 session 模式）。
- 用户偏好里有"research notebook + modern editorial + dark lab accent" 三个关键词 — 双主题给了用户三选一。

### 资料卡的"verified / need-verify / secondary / unverified-anecdote" 徽章
- 用户原始指令明确要求"对无法确认的细节，不要写成确定事实；页面中标为'需进一步核实'或改成较谨慎表述"。
- 4 种状态分别对应：直接可核（hamming/sutton/graham/feynman/olah-carter/karpathy）、可获取但版本需进一步核实（schulman/shannon/ng）、二手资料为主（darwin）、难以独立核实的私人轶事（radford）。
- 这本身就是对文中第 5 节"凝视输出"的研究能力实践 — 引用就要标核查状态。
