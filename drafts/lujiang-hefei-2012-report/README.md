## 内容说明

### 报道定位
本页面是一份**严肃非虚构报道的结构性草案**，基于公开报道与法律文本制作，用于：
- 整理已知事实与未解问题
- 规划采访方向
- 建立证据台账
- 明确报道伦理边界

### 核心原则
1. **不展示原图**：所有视觉元素均为抽象图形、流程图、时间线
2. **不追踪现状**：不公开当事人现用账号、住址、家庭成员
3. **证据分级**：A/B/C/D/E五级体系，只将A/B级作为事实依据
4. **不人肉**：不做人脸识别、身份比对

### 页面结构（v0.3 叙事型）
- **报道边界**：不展示原图、不追踪现状、不做人脸识别、只讨论公开传播与机构处理
- **第一节：旧案如何重返公共视野**：8月初流传、8月9日身份指认、8月10日处分
- **第二节：旧图如何变成公共指认**：微博传播、贴吧指认、标签附着机制
- **第三节：一次越辟越乱的辟谣**：官方通报、语义错位、引用未经核实来源
- **第四节：公开报道中的十一小时**：财新时间线、不判断程序合法性
- **第五节：责任的不对称**：被处分者 vs 消失的传播者/泄露者/错误指认者
- **第六节：证据台账**：可交互筛选的证据等级表格与原始来源缺口
- **第七节：仍未完成的追问**：8个关键问题，不诱导人肉
- **第八节：资料来源与方法说明**：来源列表与方法声明

## v0.2 → v0.3 主要变化

| 维度 | v0.2（资料展示型） | v0.3（叙事报道型） |
|------|------------------|------------------|
| 结构 | 时间线+故事线+证据台账并列 | 按叙事顺序推进 |
| 章节数 | 12个 | 9个（含报道边界） |
| 主线 | 不明显 | "照片之后"贯穿始终 |
| 叙事声音 | 弱 | 强，有明确的叙事者 |
| 重复 | 时间线与故事线重复 | 减少重复 |
| 证据展示 | 集中展示 | 分散到各章节 |
| 采访计划 | 独立章节 | 不再展示（内部保留） |

## 技术说明

- 纯静态 HTML/CSS/JS，无需构建工具
- 数据以 JSON 文件存储，便于后续维护
- SVG 图表为矢量图形，可无损缩放
- 响应式设计，支持移动端阅读
- 新增：章节导航高亮、pull quote、evidence hint、question card 样式
- LHR-1.4Style：跳过链接、焦点样式、表格横向滚动、打印样式、键盘交互支持

## 限制与声明

- **非正式发布**：本页面为本地离线草案，不公开发布
- **不完整性**：v0.3 版本基于公开资料，大量信息待采访核实
- **未采访**：未联系任何当事人、记者、机构或法律专家
- **伦理约束**：严格遵守不传播不雅内容、不公开隐私的原则
- **法律声明**：不构成法律意见，刑法条文仅作为背景
