$cat ./showcases/url-quality/README.md

每天通过 Agent 阅读大量 URL，但不同网站的反爬策略、内容结构差异极大。如何评估一次 URL 抓取是否「成功」？

这套三维评估框架把质量判断从「黑盒」变成「可观测、可记录、可迭代」。

$cat ./framework.md
维度取值说明
READ_STATUSFULL_READ / PARTIAL_READ / METADATA_ONLY / ERROR / BLOCKED正文抓取程度
content_qualityok / suspicious / poor内容质量评估
quality_notes自由文本异常原因记录
$cat ./downgrade-rules.md

自动降级规则

FULL_READ + suspicious → PARTIAL_READ

FULL_READ/PARTIAL_READ + poor → METADATA_ONLY

降级触发后，系统在 daily log 中保留 URL 和 metadata，正文标记为待人工补读。

$cat ./observation-method.md

先观察，后修改

不基于单个案例调整全局规则。所有 URL 抓取结果进入观察日志，积累足够样本后才考虑阈值调优。

基线样本覆盖：个人博客、新闻站、杂志、目录页——确保评估框架在不同场景下的稳健性。
