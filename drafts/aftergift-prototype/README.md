# 后来礼物 Aftergift — 原型

> 故事型礼物流转平台。把关系结束后难以处理的礼物，变成可以被讲述、流转、出售、交换或捐出的故事物。

## 项目简介

这是一个静态单页 Web App 原型，展示了「后来礼物」的核心交互流程和视觉风格。产品定位为「故事型礼物流转平台」——用户处理的不是单纯商品，而是关系结束或变化后的礼物，每件礼物背后都要写下一个故事。

**核心原则：** 温柔、克制、干净、有一点伤感。不鼓励报复、羞辱、猎奇或网暴。

- [x] Hero 首屏：标题 + 副标题 + 一句话介绍 + 双按钮

- [x] 礼物详情 Modal：完整故事 + 安全提示 + 按钮交互反馈
- [x] 发布礼物模拟表单：所有字段 + 情绪标签 + 字数计数 + 前端临时新增卡片
- [x] 安全与伦理说明区：禁止事项 + 鼓励方式
- [x] 产品路线图：MVP → 社区版 → 交易版 → 线下版
- [x] Toast 反馈系统
- [x] 移动端适配（桌面 / 平板 / 手机）
- [x] **筛选空状态**：无结果时显示温柔的提示文案和 CSS 信封图标
- [x] **故事模板提示**：4 个写作引导卡片（来历 / 曾经的意义 / 现在的变化 / 下一站），点击可插入提示到故事框
- [x] **故事预检功能**：点击按钮后对「一句话故事」和「完整故事」做前端本地规则检测，输出三级结果（适合公开 / 建议修改 / 不建议直接公开）

## 隐私与伦理边界

原型层面已内置以下安全设计：

- 表单收集关系类型、情绪标签等有限信息，不收集姓名、地址、手机号
- **故事预检规则引擎**：前端自动检测手机号、微信号、QQ号、邮箱、身份证格式数字、详细地址关键词、报复性表达、可识别姓名模式，输出温柔提示
- **发布前确认 Checkbox**：3 项必选确认，强制用户自我审核内容
- **故事预览**：发布前可预览他人将看到的实际样式
- **故事模板引导**：提供 4 个写作提示，引导用户聚焦礼物和自己的感受，而非描述对方
- Modal 详情页包含安全提示文案
- 禁止曝光他人真实身份信息（姓名 / 照片 / 手机 / 地址 / 公司 / 社交账号）
- 禁止发布侮辱、报复、控诉式内容
- 鼓励匿名化叙述，聚焦物品和自己的感受
- 所有示例数据均为虚构

## 故事预检规则说明

预检引擎检测以下三类风险（纯前端静态规则，不上传任何数据）：

**A. 身份信息风险**
- 手机号（1开头的11位数字组合）
- 微信号、WeChat、wx/WX 等
- QQ号
- 邮箱地址
- 身份证格式数字
- 详细地址关键词（小区、楼栋、单元、门牌等）
- 社交平台账号关键词（微博、抖音、小红书、Instagram、Twitter等）

**B. 报复性表达风险**
- 渣男、渣女、报复、曝光、挂人、人肉、骗子、毁掉等控诉类词汇
- 去死、恨死、恶心等诅咒类表达
- 贱、不要脸、无耻等侮辱类词汇

**C. 可识别关系对象风险**
- 出现「他叫」「她叫」「前任叫」「老公叫」等接真实姓名的模式
- 出现「姓名：XXX」等明确姓名格式
- 出现全名

预检结果分为三级：
- **适合公开**：无风险检测结果，且三项确认已全部勾选
- **建议修改后公开**：有任何一项风险检测匹配，但不做强制拦截
- **不建议直接公开**：有报复性或攻击性表达被检测到

## 后续可扩展方向

1. **真实后端**：用户系统 + 数据库持久化
2. **真实交易**：担保支付 + 物流追踪 + 交换撮合
3. **社区功能**：收藏故事 + 温和评论 + 匿名私信
4. **线下版**：城市礼物交换会 + 关系旧物展
5. **移动 App**：React Native 或 Flutter 版本
6. **内容审核层**：AI 辅助识别违规内容（含后端规则引擎）
7. **故事模板编辑器**：结构化引导（如：填空式模板、心情曲线记录）
8. **匿名私信系统**：不暴露真实身份的读者与发布者通信机制

---

*这是一个静态原型，所有故事均为虚构。平台不处理真实交易，不存储用户敏感信息。*
