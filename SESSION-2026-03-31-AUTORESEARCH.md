## 🎯 核心概念

### 1. Autoresearch 是什么？

> *"你睡前给 AI 代理一个目标，比如'把验证损失降到最低'，然后去睡觉。第二天早上醒来，代理已经帮你跑完了一百个实验。"*

**核心理念**: 将人类从低价值的重复劳动中解放，专注于提出问题和定义目标。

### 2. 三个文件的极简设计

```
autoresearch/
├── prepare.py      # 基础设施层（一次性设置）
├── train.py        # 实验沙盒（代理可修改）
└── program.md      # 人类-AI 沟通接口
```

**设计哲学**: 约束带来聚焦
- 只能修改 train.py → 避免在无关紧要处浪费时间
- 5 分钟时间限制 → 强制快速迭代
- train.py 足够完整 → 代理可以进行任何有意义的实验

### 3. 棘轮式优化

```
提出假设 → 修改代码 → 运行实验（5分钟）→ 评估指标
    ↑                                              │
    └────── 改进？保留 : 回滚 ──────────────────────┘
```

- **速度**: 每小时 12 次实验
- ** overnight**: ~100 次迭代
- **Git**: 记忆系统（每次实验是一个 commit）

## 🔬 技术亮点

### 与传统超参数调优的区别

| 维度 | Optuna/Ray Tune | Autoresearch |
|-----|-----------------|--------------|
| **搜索空间** | 人类预定义 | LLM 想象力和创造力 |
| **可修改内容** | 超参数 | 任意代码 |
| **学习能力** | 贝叶斯优化 | 从每次实验学习 |
| **创新程度** | 参数组合 | 架构创新 |

### AI 代理可以做什么

**架构层面**:
- 重新设计注意力机制
- 修改激活函数
- 调整层归一化位置

**优化层面**:
- 实验不同优化器
- 改变学习率调度
- 调整批量大小

**创新示例**: 代理发现的一些改进是人类研究者可能不会想到的细节优化。

## 💡 核心洞察

### 1. 新范式：Agentic Engineering

> *"工程师不再直接编写 99% 的代码，而是编排代理并充当监督者。"*

**角色转变**:
```
传统工程师: 执行者 → 写代码、调参数、跑实验
Agentic Engineer: 战略制定者 → 定义目标、编排代理、监督结果
```

### 2. 人人都可以是深度学习工程师

> *"你不需要博士学位，不需要昂贵的 GPU 集群，甚至不需要深入理解每一个技术细节。你只需要有好的问题、清晰的目标，以及一台能运行代码的电脑。"*

**门槛降低的三要素**:
1. AI 代理自主研究
2. 一台 MacBook 就能运行
3. 全球计算资源协作

### 3. Vibe Coding 的延伸

```
Vibe Coding: AI 帮你写代码
    ↓
Autoresearch: AI 帮你做研究
    ↓
未来: AI 帮你解决任何问题
```

## ⚠️ 批判性反思

### 潜在风险

1. **黑盒问题**: 代理找到的优化可能难以理解
2. **局部最优陷阱**: 代理可能在局部最优附近反复探索
3. **过度自动化**: 研究者可能失去对技术的深入理解

### 适用边界

**适合**:
- 超参数搜索空间大的问题
- 需要大量试错的架构探索
- 有明确评估指标的任务

**不适合**:
- 需要深度理论创新的研究
- 评估指标难以量化的任务
- 对可解释性要求极高的场景

## 🛠️ 实践建议

### 开始使用

```bash
# 1. 克隆仓库
git clone https://github.com/karpathy/autoresearch.git

# 2. 运行 prepare.py（一次性）
python prepare.py

# 3. 编辑 program.md 定义目标

# 4. 启动代理
python run_agent.py
```

### 好的研究目标

- ✅ "在 5 分钟内将验证损失降低到 2.0 以下"
- ✅ "尝试找到比当前 baseline 更好的注意力机制"
- ❌ "让模型更好"（太模糊）
- ❌ "达到 SOTA 性能"（不切实际）
