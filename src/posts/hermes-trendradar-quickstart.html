5 分钟快速集成

步骤 1: 部署 TrendRadar (2 分钟)

<h1>1. Fork 项目到你的 GitHub</h1>


<h1>访问: https://github.com/sansan0/TrendRadar</h1> <h1>点击 Fork 按钮</h1>



<h1>2. 启用 GitHub Actions</h1> <h1>在你的 Fork 中，点击 Actions → 启用 Actions</h1>



<h1>3. 配置密钥</h1> <h1>Settings → Secrets → New repository secret</h1> <h1>添加: API_KEY (你的 OpenAI API Key)</h1>



<h1>4. 完成！</h1> <h1>等待 1-2 分钟，自动运行</h1> <h1>查看结果: https://yourname.github.io/TrendRadar</h1>

步骤 2: 添加 Hermes Skill (2 分钟)

<h1>1. 复制 Skill 文件</h1>


cp ~/research/hermes-trendradar-integration/skills/trendradar_skill.py \    ~/hermes-agent/skills/



<h1>2. 添加到 Hermes 配置</h1> echo "trendradar" >> ~/hermes-agent/config/enabled_skills.txt



<h1>3. 重启 Hermes</h1> hermes restart

故障排除

问题 1: TrendRadar 数据未更新

检查: 
<h1>查看 GitHub Actions 运行状态</h1>


<h1>https://github.com/yourname/TrendRadar/actions</h1>



<h1>检查是否配置了 API_KEY</h1> <h1>Settings → Secrets → API_KEY</h1>
解决: 
<h1>手动触发运行</h1>


<h1>Actions → TrendRadar Workflow → Run workflow</h1>

问题 2: Hermes 无法读取数据

检查: 
<h1>检查缓存文件</h1>


ls -la ~/.hermes/trendradar/



<h1>查看日志</h1> hermes logs | grep trendradar
解决: 
<h1>清除缓存重启</h1>


rm ~/.hermes/trendradar/hot_topics_cache.json hermes restart
