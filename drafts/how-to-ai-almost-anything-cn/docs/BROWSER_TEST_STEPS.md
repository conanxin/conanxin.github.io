# 浏览器人工测试步骤
# How2AI 中文课程页 — Phase 4

**适用浏览器**: Chrome / Firefox / Safari（桌面版）
**测试设备**: 桌面电脑，分辨率 1280×800 或以上
**预计时间**: 15–20 分钟

---

## 准备工作

### Step 0：启动本地静态服务器

打开终端，执行：

```bash
cd ~/conanxin.github.io/drafts/how-to-ai-almost-anything-cn
python3 -m http.server 8080
```

服务器启动后，保持终端窗口打开。

> 如果端口 8080 被占用：
> ```bash
> python3 -m http.server 8081
> ```
> 访问 `http://localhost:8081/` 代替。

---

### Step 1：打开页面

在浏览器地址栏输入：

```
http://localhost:8080/
```

或（如果用 8081）：

```
http://localhost:8081/
```

确认页面加载，Hero 区域可见。

---

## 逐项测试

### Step 2：Hero 区检查

1. 确认看到紫色渐变背景 + 神经网络 SVG 动画节点
2. 确认标题 "How to AI (Almost) Anything" 清晰可见
3. 确认副标题："把 AI 接入真实世界的多模态、多感官与交互系统"
4. 确认三个按钮：
   - 🚀 开始学习
   - 📚 查看官方阅读清单
   - 📖 七角色论文读法
5. 确认统计数据：27 节课程 / 5 大模块 / 32 精选导读 / 85+ 官方文献
6. 确认进度区显示 "0/23" 和 "0/32"（空白状态）

---

### Step 3：模块 Tab 切换

1. 滚动到顶部 Nav Tab 区域（sticky，背景半透明）
2. 点击 **Module 0 · 导论** → 应只显示 2 个 session
3. 点击 **Module 1 · AI 基础** → 应只显示约 4 个 session
4. 点击 **Module 2 · 多模态 AI** → 应只显示约 7 个 session
5. 点击 **Module 3 · 大模型** → 应只显示约 6 个 session
6. 点击 **Module 4 · 交互式 AI** → 应只显示约 4 个 session
7. 点击 **全部课程** → 应显示全部 session

每步确认：时间线（左侧）是否同步过滤。

---

### Step 4：Session 卡片展开/折叠

1. 点击第一个 **Week 1 · Lecture** 的卡片 header（"▼" 图标那一行）
2. 确认卡片展开，显示详细内容
3. 确认显示：中文摘要 / 关键概念标签 / 学习目标 / 思考题
4. 点击第二张卡片 header
5. 确认第一张卡片自动折叠（只保留一张展开）
6. 关闭：再点击第一张卡片的 header，确认为折叠状态

---

### Step 5：勾选 Checklist

1. 展开第一节 Lecture 卡片
2. 找到 "标记为已完成" checkbox，**勾选**
3. 确认：卡片 header 旁出现 ✅ 标记
4. 滚动到 Hero 区，确认进度从 "0/23" 变为 "1/23"
5. 确认进度条百分比更新

---

### Step 6：写一条笔记

1. 展开第一节 Lecture 卡片
2. 点击 **📝 学习笔记** 按钮
3. 确认弹出笔记 Modal（覆盖层）
4. 确认顶部显示本节中文摘要
5. 在文本域输入一行文字，例如："今天学习了多模态 AI 的定义"
6. 点击 **💾 保存笔记**
7. 确认出现 toast 提示："笔记已保存 💾"
8. 点击 × 关闭 modal

---

### Step 7：刷新页面，确认 localStorage 保留

1. 按 **Cmd+R**（Mac）或 **Ctrl+R**（Windows）刷新页面
2. 确认：
   - [ ] 第一节卡片的 checkbox 仍然是勾选状态
   - [ ] Hero 进度仍显示 "1/23"
   - [ ] 笔记 modal 中输入的文字仍然存在
3. 这证明 localStorage 功能正常

---

### Step 8：导出 Markdown

1. 展开第一节 Lecture 卡片
2. 点击 **📝 学习笔记**
3. 点击 **📤 导出为 Markdown**
4. 确认：浏览器弹出下载，或新标签页显示 Markdown 内容
5. 确认内容包含：课程标题 / 导出时间 / 本节笔记文字

---

### Step 9：切换精选阅读 / 官方阅读

1. 滚动到 **#05 论文导航** section
2. 确认显示 **精选中文导读** 标签页（默认）
3. 确认有 32 张阅读卡片
4. 点击 **Foundation** filter → 应只显示 Foundation 分类的卡片
5. 点击 **📋 官方完整阅读清单** 按钮
6. 确认切换到官方阅读（灰色底色区）
7. 确认有 85+ 张卡片
8. 点击 **核心阅读** filter → 确认只显示核心阅读卡片
9. 点击 **精选中文导读** 按钮切回

---

### Step 10：修改一篇论文的阅读状态

1. 在精选阅读区，随便点开一张阅读卡片
2. 找到四个状态按钮：**待读 / 在读 / 已读 / 跳过**
3. 点击 **在读** 按钮（黄色高亮）
4. 确认出现 toast："阅读状态已更新"
5. 刷新页面
6. 确认该卡片的"在读"状态仍然保持（localStorage 保留）
7. 确认 Hero 阅读进度从 "0/32" 变为 "1/32"

---

### Step 11：术语表搜索

1. 滚动到 **#09 术语表** section
2. 在搜索框输入：`multimodal`（英文）
3. 确认只显示包含 "multimodal" 的术语卡片
4. 清空搜索框，输入：`对齐`（中文）
5. 确认只显示包含"对齐"的术语卡片
6. 清空，确认显示全部 47 条术语

---

### Step 12：七角色论文读法

1. 点击 Hero 区的 **📖 七角色论文读法** 按钮
2. 页面 smooth scroll 到 #06 section
3. 确认看到七角色 SVG 轮盘（彩色扇区）
4. 点击 **Archaeologist** 角色按钮
5. 确认切换到考古视角详情卡片（右侧内容区）
6. 点击 **Hacker** 角色按钮
7. 确认切换到黑客视角详情卡片
8. 逐个点击剩余 5 个角色，确认每个都有独特内容

---

### Step 13：浏览页面底部

1. 滚动到页面最底部（footer）
2. 确认版权声明存在："本页面为基于 MIT MAS.S60 公开课程资料整理的中文学习导览，不代表 MIT 官方中文版本"
3. 确认有"关于链接状态"段落（解释 MIT slides / YouTube / arXiv 的各自情况）
4. 确认外部链接指向正确的页面（官方课程 / MIT OCW / Paul Liang / YouTube）

---

### Step 14：移动端模拟（可选）

1. 打开 Chrome 开发者工具（F12）
2. 按 **Cmd+Shift+M**（Mac）或 **Ctrl+Shift+M**（Windows）切换到移动端模拟
3. 选择 **iPhone 12** 或设置宽度为 **390px**
4. 确认：
   - [ ] 页面无横向滚动条
   - [ ] Hero 按钮是否堆叠（不横向并排）
   - [ ] Nav tabs 是否有滚动条
   - [ ] Session 卡片是否可读
   - [ ] 阅读卡片是否变为单列
5. 尝试点击一个 session 卡片，确认展开/折叠正常
6. 点击"学习笔记"按钮，确认 modal 正常弹出
